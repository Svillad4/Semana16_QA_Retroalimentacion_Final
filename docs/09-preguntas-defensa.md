# 09 - Preguntas para defensa del proyecto

Prepara respuestas para estas preguntas antes de la exposicion final.

1. Que problema resuelve tu proyecto? 
   Resuelve la necesidad de registrar, organizar y justificar bugs y mejoras en un proyecto web final.
2. Cual es la funcionalidad principal? 
   La funcionalidad principal es registrar bugs, marcar correcciones y documentar mejoras para la entrega.
3. Que tecnologias usaste y por que? 
   Use HTML, CSS, JavaScript, Bootstrap, SweetAlert2, Chart.js y JSON Server para crear una experiencia sencilla y ordenada.
4. Como comprobaste que JSON Server funciona? 
   Lo verifique abriendo la ruta de la API y confirmando que devolvia datos del archivo db.json.
5. Que bug fue mas importante y como lo corregiste? 
   El bug mas importante fue la eliminacion sin confirmacion, y lo corregi con una alerta de confirmacion.
6. Que evidencia demuestra que aplicaste correcciones? 
   La documentacion en los archivos de docs y los cambios hechos en el codigo del proyecto.
7. Que retroalimentacion recibiste? 
   La retroalimentacion fue que era importante mejorar la validacion y la seguridad de las acciones.
8. Que mejora futura implementarias primero? 
   Implementaria la exportacion de reportes en CSV para organizar mejor la informacion.
9. Como mejoraste la usabilidad o la claridad del proyecto? 
   Mejoré los mensajes, agregue confirmaciones y ordene mejor la informacion del dashboard.
10. Que aprendiste del proceso de QA y mejora continua? 
   Aprendí que revisar, probar y documentar un proyecto es tan importante como construirlo.
