# 10 - Guia profunda de clase

## Semana 16: QA, retroalimentacion y mejora continua

En esta actividad trabajaras como si estuvieras cerrando un proyecto real. El objetivo no es escribir mas codigo sin orden, sino revisar la calidad de lo que ya existe, detectar errores, priorizarlos, corregirlos y demostrar que el proyecto mejoro.

## Que es QA?

QA significa aseguramiento de calidad. En un proyecto web, QA consiste en revisar si el sitio cumple con lo que promete: si abre correctamente, si los botones funcionan, si los datos se guardan, si los mensajes son claros, si el diseno es usable y si la documentacion permite que otra persona entienda el proyecto.

## Que es un bug?

Un bug es un error, defecto o comportamiento incorrecto dentro del proyecto. Puede ser tecnico, visual, funcional o documental. Por ejemplo:

- Un formulario que permite datos vacios.
- Un boton que no funciona.
- Una ruta de API incorrecta.
- Un README incompleto.
- Una captura de evidencia faltante.
- Un diseno que se rompe en celular.

## Por que priorizar bugs?

No todos los errores tienen el mismo impacto. Algunos impiden que el proyecto funcione y deben corregirse primero. Otros son detalles visuales que pueden esperar. Priorizar ayuda a tomar decisiones tecnicas.

## Que es coevaluacion?

La coevaluacion es la revision entre pares. Un companero o equipo revisa tu proyecto y te da retroalimentacion. Esta retroalimentacion debe ser tecnica, respetuosa y util.

## Que es mejora continua?

La mejora continua significa reconocer que el proyecto puede seguir evolucionando. Aunque funcione, puede mejorar en rendimiento, accesibilidad, seguridad, documentacion o nuevas funcionalidades.

## Como se conecta con la exposicion final?

Durante la exposicion, no basta con mostrar la pagina. Debes explicar que revisaste, que errores encontraste, que corregiste, que evidencia tienes y que mejoras propones. Esto demuestra dominio del proyecto.
